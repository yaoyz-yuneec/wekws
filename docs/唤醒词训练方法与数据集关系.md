# 唤醒词训练方法与数据集关系

本文档说明 WeKws 中数据集与训练方法的对应关系，帮助理解不同场景下如何选择合适的训练策略。

---

## 1. 概述

WeKws 的训练流程可以概括为：

```
原始音频数据集
      │
      ▼
  Kaldi 格式文件 (wav.scp + text)
      │
      ▼
  data.list (JSON 行格式)
      │
      ▼
  模型训练 (特征提取 → 时序建模 → 分类输出)
      │
      ▼
  模型导出 (JIT / ONNX)
      │
      ▼
  运行时推理 (Python / C++ / Android)
```

不同的数据集在**标签定义**、**损失函数**、**模型结构**三个方面有不同的配置。理解这些差异是训练自定义唤醒词的关键。

---

## 2. 数据集与训练模式的对应关系

WeKws 目前支持三类数据集，分别对应三种典型的训练范式：

### 2.1 Speech Commands — 多关键词分类

| 项目 | 说明 |
|------|------|
| **数据集** | Google Speech Commands v0.01/v1 |
| **场景** | 30 个关键词中识别 10 个目标词（如 yes/no/up/down/on/off/go/stop/left/right） |
| **标签映射** | `<UNKNOWN> 0`, `<YES> 1`, ..., `<GO> 10`（共 11 类） |
| **num_keywords** | `11` |
| **损失函数** | `ce`（交叉熵） |
| **Classifier** | `GlobalClassifier`（整句平均池化后分类） |
| **Activation** | `nn.Identity`（输出 logits，由 CE loss 处理） |
| **推理方式** | 整句分类，输出 11 个类别的概率分布 |
| **参考配置** | `examples/speechcommand_v1/s0/conf/mdtc.yaml` |

数据流向：

```
data/local/speech_commands_v1/{train,valid,test}/
    │  每个关键词类别一个子目录，内含 .wav 文件
    │
    [prepare_speech_command.py] 分配标签：
    │  - 10 个目标词 → 对应 <YES>/<NO>/... 等词符
    │  - 其余 20 个词 → <UNKNOWN>
    ▼
data/{train,valid,test}/text
    │  on_xxx <ON>
    │  yes_yyy <YES>
    │  bed_zzz <UNKNOWN>
    │
    [make_list.py] → data.list
    ▼
训练: --num_keywords 11 --dict ./dict
    dict.txt:
      <UNKNOWN> 0
      <YES>     1
      <NO>      2
      ...
      <GO>      10
```

### 2.2 Hey Snips — 单唤醒词二分类

| 项目 | 说明 |
|------|------|
| **数据集** | Snips Wake Word 数据集 |
| **场景** | 检测是否说了 "Hey Snips"（单唤醒词） |
| **标签映射** | `<FILLER> -1`, `<HEY_SNIPS> 0` |
| **num_keywords** | `1` |
| **损失函数** | `max_pooling`（默认，在 executor.py 中指定） |
| **Classifier** | `LinearClassifier`（逐帧输出） |
| **Activation** | `nn.Sigmoid`（每个关键字输出 0~1 概率） |
| **推理方式** | 流式帧级别检测，支持滑动窗口 |
| **参考配置** | `examples/hey_snips/s0/conf/ds_tcn.yaml` |
| **增强配置** | reverb_prob=0.2, noise_prob=0.3 |

数据流向：

```
data/local/hey_snips_research_6k_en_train_eval_clean_ter/
    ├── audio_files/         # .wav 音频文件
    ├── train.json           # 标注：is_hotword=1 或 0
    ├── dev.json
    └── test.json
    │
    [prepare_data.py] 读取 JSON，按 is_hotword 分配标签：
    │  is_hotword=1 → <HEY_SNIPS> (ID=0)
    │  is_hotword=0 → <FILLER>    (ID=-1)
    ▼
data/{train,dev,test}/text
    │  utt_001 <HEY_SNIPS>
    │  utt_002 <FILLER>
    │
    [make_list.py] → data.list
    ▼
训练: --num_keywords 1 --dict ./dict
    dict.txt:
      <FILLER> -1       ← 负数表示非关键词
      <HEY_SNIPS> 0     ← 非负数表示关键词
```

### 2.3 Mobvoi / Hi Xiaowen — 多唤醒词检测

| 项目 | 说明 |
|------|------|
| **数据集** | Mobvoi 唤醒词数据集（中文） |
| **场景** | 检测 "嗨小问" 或 "你好问问"（多唤醒词） |
| **标签映射** | `<FILLER> -1`, `<HI_XIAOWEN> 0`, `<NIHAO_WENWEN> 1` |
| **num_keywords** | `2` |
| **损失函数** | `max_pooling` 或 `ctc`（两套配置） |
| **Classifier** | `LinearClassifier` + `Sigmoid`（max_pooling）或 `nn.Identity`（CTC） |
| **推理方式** | 流式检测（CTC 方案支持 prefix beam search 解码） |
| **参考配置** | `examples/hi_xiaowen/s0/conf/ds_tcn.yaml`（max_pooling） |
| | `examples/hi_xiaowen/s0/conf/fsmn_ctc.yaml`（CTC） |

## 3. 标签系统详解

### 3.1 dict.txt 的规则

```
<词符> ID
```

- **ID >= 0**：目标任务词（关键词），模型需要学习识别这些词
- **ID < 0**（通常为 -1）：非关键词（Filler/UNKNOWN），模型需要学会忽略

示例：

```
# Speech Commands（多分类）
<UNKNOWN> 0   ← 非目标词（填充类）
<YES>     1   ← 目标词
<NO>      2   ← 目标词
...

# Hey Snips（单唤醒词）
<FILLER> -1    ← 非关键词
<HEY_SNIPS> 0  ← 目标词

# Hi Xiaowen（多唤醒词）
<FILLER> -1        ← 非关键词
<HI_XIAOWEN> 0     ← 目标词 1
<NIHAO_WENWEN> 1   ← 目标词 2
```

### 3.2 text 文件的标签映射

`prepare_data.py` 脚本负责将原始标注转为 `text` 文件中的词符。`wekws/bin/train.py` 中的 `CharTokenizer` 会将词符按 `dict.txt` 映射为数字 ID，最终传给损失函数。

## 4. 模型结构详解

### 4.1 统一架构（KWSModel）

所有训练范式共享同一模型结构：

```
输入: Fbank/MFCC 特征 (T × D)
      │
      ▼
┌─────────────────┐
│  GlobalCMVN     │  全局均值方差归一化（可选）
├─────────────────┤
│  Preprocessing  │  特征维度投影 + 下采样
│                 │  - LinearSubsampling1
│                 │  - Conv1dSubsampling1
│                 │  - NoSubsampling
├─────────────────┤
│  Backbone       │  核心时序建模网络
│                 │  - TCN (时序卷积网络)
│                 │  - MDTC (多尺度时序卷积)
│                 │  - GRU (门控循环单元)
│                 │  - FSMN (前馈序列记忆网络)
├─────────────────┤
│  Classifier     │  分类输出层
│                 │  - LinearClassifier: 逐帧线性输出 + Sigmoid
│                 │  - GlobalClassifier: 全局平均池化 + 全连接 + Identity
│                 │  - LastClassifier: 取最后一帧 + 全连接
└─────────────────┘
      │
      ▼
输出: 关键词概率分布
```

### 4.2 Classifier 与 Activation 的选择

| 组合 | backnone + classifier | activation | 损失函数 | 适用场景 |
|------|----------------------|-----------|---------|---------|
| **唤醒词模式** | backbone + `LinearClassifier` | `nn.Sigmoid` | `max_pooling` | 单/多唤醒词流式检测 |
| **命令词模式** | backbone + `GlobalClassifier` | `nn.Identity` | `ce`（交叉熵） | 语音命令短句分类 |
| **CTC 模式** | backbone + `LinearClassifier` 或 `nn.Identity` | `nn.Identity` | `ctc` | 流式关键词检测+转录 |

**判断依据**（来自 `wekws/model/kws_model.py`）：
- 如果配置中包含 `classifier` 字段 → `GlobalClassifier` / `LastClassifier` / `nn.Identity`，使用 `nn.Identity` 作为 activation
- 如果不包含 `classifier` 字段 → 使用 `LinearClassifier` + `nn.Sigmoid`

```yaml
# 唤醒词模式（无 classifier 字段）
model:
    hidden_dim: 64
    preprocessing:
        type: linear
    backbone:
        type: tcn
        ds: true
        num_layers: 4
# 默认使用 LinearClassifier + Sigmoid

# 命令词模式（有 classifier 字段）
model:
    hidden_dim: 64
    preprocessing:
        type: linear
    backbone:
        type: mdtc
        num_stack: 4
    classifier:
        type: global       # global / last / identity
        dropout: 0.5
# 使用 GlobalClassifier + Identity + cross_entropy
```

### 4.3 损失函数的选择

损失函数在 `wekws/utils/executor.py` 中通过 `training_config` 的 `criterion` 字段指定：

```yaml
# 在 conf/*.yaml 的 training_config 中设置
training_config:
    criterion: ce            # 交叉熵（用于 Speech Commands）
    # 或
    criterion: max_pooling   # 最大池化损失（用于唤醒词，默认值）
    # 或
    criterion: ctc           # CTC 损失（用于流式关键词）
```

#### ce（交叉熵）

```
适用: 整句多分类（Speech Commands）
特点:
  - 对整个句子输出单个类别概率分布
  - 配合 GlobalClassifier（整句平均池化）
  - 简单高效，适合短句命令识别
```

#### max_pooling（最大池化损失）

```
适用: 唤醒词检测（Hey Snips / Hi Xiaowen）
特点:
  - 正样本（关键词）: 取所有帧中最高概率帧 → -log(max_prob)
    含义: 只要有一帧检测到关键词，就算正确
  - 负样本（非关键词）: 取最难帧（概率最高的错误帧） → -log(1 - max_prob)
    含义: 所有帧都不能误触发
  - 支持流式推理
```

#### ctc（CTC 损失）

```
适用: 流式唤醒词 + 语音转录（Hi Xiaowen CTC 方案）
特点:
  - 支持 CTC prefix beam search 解码
  - 可以同时检测关键词和转录语音内容
  - 配合 FSMN 等流式 backbone
  - 支持更复杂的多词唤醒词
```

## 5. 训练流程分阶段详解

以 `examples/speechcommand_v1/s0/run.sh` 为例，完整的训练流程包含 5 个阶段：

### Stage -1: 数据下载与提取

```bash
local/data_download.sh --dl_dir ./data/local
# 从 TensorFlow 下载 speech_commands_v0.01.tar.gz 到 data/local/

python local/split_dataset.py ./data/local/speech_commands_v1
# 将原始音频按官方划分为 train/valid/test 三份
```

### Stage 0: 生成 Kaldi 格式文件

```bash
# 1. 创建字典
echo "<UNKNOWN> 0" > dict/dict.txt
echo "<YES> 1" >> dict/dict.txt
...

# 2. 对每个子集 (train/test/valid) 生成 wav.scp + text
find $speech_command_dir/$x -name "*.wav" > data/$x/wav.list
python local/prepare_speech_command.py --wav_list=data/$x/wav.list --data_dir=data/$x
```

### Stage 1: 特征预处理

```bash
# 1. 计算全局 CMVN 统计量（均值方差归一化）
tools/compute_cmvn_stats.py --train_config $config \
  --in_scp data/train/wav.scp \
  --out_cmvn data/train/global_cmvn

# 2. 计算音频时长 + 生成 data.list（JSON 行格式）
tools/wav_to_duration.sh data/$x/wav.scp data/$x/wav.dur
tools/make_list.py data/$x/wav.scp data/$x/text data/$x/wav.dur data/$x/data.list
```

`data.list` 的 JSON 行格式示例：

```json
{"key": "on_71f9bba8_nohash_0", "txt": "<ON>", "duration": 1.0, "wav": "./data/local/speech_commands_v1/train/on/71f9bba8_nohash_0.wav"}
```

### Stage 2: 模型训练

```bash
torchrun --standalone --nnodes=1 --nproc_per_node=$num_gpus \
  wekws/bin/train.py \
    --config $config \
    --train_data data/train/data.list \
    --cv_data data/valid/data.list \
    --model_dir $dir \
    --num_keywords $num_keywords \
    --min_duration 50 \
    --dict ./dict \
    --cmvn_file data/train/global_cmvn \
    --norm_var
```

训练内部流程（`train.py`）：

```
1. 读取 data.list → 加载音频 → waveform
2. 重采样到 16kHz
3. 提取 Fbank/MFCC 特征
4. CharTokenizer 将 "<ON>" 映射为数字 ID (如 7)
5. 模型前向传播 → logits
6. 计算损失 (ce/max_pooling/ctc)
7. 反向传播更新参数
```

### Stage 3: 模型平均与评估

```bash
# 取验证集 loss 最低的 N 个 checkpoint 平均
python wekws/bin/average_model.py --dst_model avg_10.pt --src_path $dir --num 10 --val_best

# 测试
python wekws/bin/compute_accuracy.py --test_data data/test/data.list --checkpoint avg_10.pt
```

### Stage 4: 模型导出

```bash
# JIT 模型（Python 推理）
python wekws/bin/export_jit.py --config $dir/config.yaml --checkpoint avg_10.pt --jit_model model.zip

# ONNX 模型（C++/Android/边缘设备）
python wekws/bin/export_onnx.py --config $dir/config.yaml --checkpoint avg_10.pt --onnx_model model.onnx
```

## 6. 如何选择训练方案

### 场景一：单唤醒词检测（如 "Hey Snips"、"旺达旺达"）

| 选择 | 推荐值 |
|------|--------|
| **参考示例** | `hey_snips/s0` |
| **标签** | `<FILLER> -1`, `<KEYWORD> 0` |
| **num_keywords** | `1` |
| **损失函数** | `max_pooling`（默认） |
| **模型结构** | backbone + `LinearClassifier` + `Sigmoid` |
| **数据增强** | reverb + noise（弥补正样本不足） |

### 场景二：多唤醒词检测（如 "嗨小问"/"你好问问"）

| 选择 | 推荐值 |
|------|--------|
| **参考示例** | `hi_xiaowen/s0` 的 `run.sh` |
| **标签** | `<FILLER> -1`, `<KWD1> 0`, `<KWD2> 1`, ... |
| **num_keywords** | 关键词个数 |
| **损失函数** | `max_pooling` |
| **模型结构** | backbone + `LinearClassifier` + `Sigmoid` |

### 场景三：流式唤醒词 + 语音转录

| 选择 | 推荐值 |
|------|--------|
| **参考示例** | `hi_xiaowen/s0` 的 `run_fsmn_ctc.sh` |
| **标签** | 完整的 token 序列 |
| **num_keywords** | 词典大小 |
| **损失函数** | `ctc` |
| **模型结构** | FSMN + `nn.Identity` |
| **推理方式** | CTC prefix beam search 解码 |

### 场景四：语音命令分类（如 Speech Commands）

| 选择 | 推荐值 |
|------|--------|
| **参考示例** | `speechcommand_v1/s0` |
| **标签** | `<UNKNOWN> 0`, `<CMD1> 1`, ... |
| **num_keywords** | 类别总数（含 UNKNOWN） |
| **损失函数** | `ce` |
| **模型结构** | backbone + `GlobalClassifier` + `Identity` |
| **推理方式** | 整句分类（非流式） |

## 7. 快速参考：主要参数与配置文件对照

| 参数 | 意义 | Speech Commands | Hey Snips | Hi Xiaowen |
|------|------|----------------|-----------|------------|
| `num_keywords` | 输出类别数（含 filler/unknown） | 11 | 1 | 2 |
| `criterion` | 损失函数类型 | `ce` | `max_pooling` | `max_pooling` / `ctc` |
| `min_duration` | 关键词最少帧数 | 50 | 50 | 50 |
| classifier | 分类器类型 | `global` | 无（使用 LinearClassifier） | 无 |
| activation | 输出激活函数 | `Identity` | `Sigmoid` | `Sigmoid` / `Identity` |
| backbone | 主干网络 | MDTC | DS-TCN | DS-TCN / FSMN |

## 8. 自定义唤醒词训练清单

如果要从零训练一个自定义唤醒词（如 "旺达旺达"），需要准备：

- [ ] **正样本音频**：至少数百条 "旺达旺达" 的录音（16kHz, 16bit, 单声道 WAV）
- [ ] **负样本音频**：大量不包含 "旺达旺达" 的随意语音
- [ ] **数据集划分**：train / valid / test 三份
- [ ] **dict/dict.txt**：`<FILLER> -1` + `<WANGDA_WANGDA> 0`
- [ ] **配置文件**：参照 `hey_snips` 的 `ds_tcn.yaml`，去掉数据增强相关配置（如果没有增强数据）
- [ ] **数据增强**（可选）：准备混响和噪声的 LMDB 数据，通过 `--reverb_lmdb` 和 `--noise_lmdb` 传入
- [ ] **训练命令**：参照 `hey_snips` 的 `run.sh`，将 `--num_keywords` 设为 1
- [ ] **评估**：计算 FAR（错误接受率）和 FRR（错误拒绝率）
- [ ] **导出**：导出 ONNX 模型部署到目标设备
